# 盘前 Checklist (每日 09:25 前过一遍)

> 基于 01-04 已学铁律 + 06-29 首战教训产出. 目的: 防盘中执行变形.
> 06-29 错误: 创业板浅止损割日内低点 / 芯片第二刀偏早 / 瞎猜东财成本.
> 盘前过一遍 = 把"知道"变成"做到". 漏一项 = 当日不操作.

## 1. 账户与成本 (铁律: 报数前必查db)
- [ ] 每持仓跑 `log cost <code>` — 真实成本, 不瞎猜
- [ ] 跑 `risk_metrics` — 组合HEAT / 板块集中度 / 最大单仓
- [ ] 跑 `market_regime` — 派发日数 / FTD / 风险等级 (NORMAL/CAUTION/HIGH/SEVERE)
- [ ] 跑 `sentiment` — 情绪温度 (<40谨慎, ≥40中性)

## 2. 市场结构 → 今日基调
- [ ] NORMAL → 可进攻 (开新仓OK)
- [ ] CAUTION + 情绪<40 → 防御 (不开新仓, 只减)
- [ ] HIGH/SEVERE → 防御 (只减不加, 考虑清仓)
- [ ] 查 close_scan 昨日 industry_flow — 资金流出板块不碰 ([J]假设)

## 3. 持仓触发位定形
- [ ] 每持仓: 支撑/阻力/回踩位具体价格 (写下来)
- [ ] 止损位: ETF 用 ATR (宽基 ≥4%, 行业 ETF ≥5%), **禁 <2% 浅止损** ([D]铁律)
- [ ] 个股用结构位 (跌破前日低**且收阴**才动)
- [ ] 强势板块留底仓 ≥30% ([E]铁律)

## 4. 今日计划 (写 `log plan`)
- [ ] 加仓候选 + 触发价 + 仓位% (凯利 sizer 算)
- [ ] 减仓计划 (减在急拉, 不减在回踩)
- [ ] 入场框架分清: 强势票形态确认 / 弱势票等支撑 ([A]假设)

## 5. 心理准备 (Mark Douglas 概率思维)
- [ ] 默认: 每笔交易是**概率**, 不是确定性
- [ ] 接受: 任何单笔结果不影响系统期望 (样本够才判定)
- [ ] 纪律优先: 计划 > 盘中情绪

## 6. 禁止区 (铁律)
- [ ] ❌ 盘中急杀恐慌卖 (06-29 创业板割日内低点踏空 224元)
- [ ] ❌ 09:30-10:00 减仓 (情绪极端区, [F]铁律)
- [ ] ❌ 日内低点卖 (等10:00后 + 破结构位确认)
- [ ] ❌ 浅止损 <2% ([D]数学bug)
- [ ] ❌ 追 moneyflow_surge + 已涨 >5% ([B]假设, 滞后信号)
- [ ] ❌ 瞎猜成本不查 db ([G]铁律)
- [ ] ❌ 减在回踩 (回踩是加仓点不是卖点, 格兰维尔第二买点)

## 7. 盘中执行口诀
- 早盘急杀 → 等 10:00 后, 看是否破结构位, 没破 = 噪音持有
- 急拉 → 减仓点 (强势留底仓≥30%)
- 回踩缩量不破 → 加仓点 / 持有
- 放量破位 → 真破位 (缩量破位=洗盘)
