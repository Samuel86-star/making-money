# 交易执行技能

> 用途: 理财顾问CFO交易执行知识库, 对照2026-06-29实战三错固化。所有阈值面向A股T+1、ETF+个股混合组合。
> 参考书目: 《海龟交易法则》Curtis Faith、《股票作手回忆录》Lefèvre、《通向金融王国的自由之路》Van Tharp、《交易心理分析》Mark Douglas、Wilder ATR原始定义。

## 一、止损设计

### 核心概念

止损的本质是**预先界定每笔交易的1R风险** (R=入场到止损的距离), 不是"亏多少受不了就砍"。三种主流方法:

| 方法 | 公式 | 优点 | 缺点 | 适用 |
|---|---|---|---|---|
| 固定百分比 | 入场价×(1−k%) | 简单 | 无视波动, 易被噪音洗 | 低波动品种/新手 |
| 波动率(ATR) | 入场价−m×ATR(14) | 自适应: 高波动放宽、低波动收紧 | 需算ATR | 趋势跟踪、ETF主力方法 |
| 结构位 | 跌破前日低/MA20/趋势线 | 反映真实供需结构 | 主观、需收盘确认 | 中线、避免假破位 |

**关键原理(ATR/Wilder)**: ATR把止损变成"自调节风险限制"——波动大时止损自动放宽, 避免被正常震荡洗出; 波动小时自动收紧, 控制风险。海龟法则用 **2×ATR(N) 作为止损距离**, 单元头寸按账户1%风险反推, 即 1单元 = 账户1% / (N×乘数)。这意味着**止损宽度与仓位成反比**: 止损宽就少买, 止损窄才多买, 总风险恒定1%。

**Van Tharp的R乘数**: 出场决定R分布, 止损决定1R大小。"随机入场+好出场"也能盈利——**出场比入场重要**。建议初始风险=账户1%。

### A股ETF具体阈值建议 (核心)

ETF日均振幅是止损阈值的硬约束——**止损线必须落在日均振幅之外, 否则注定被洗**:

| 品种类别 | 日均振幅 | 建议止损 | 示例 |
|---|---|---|---|
| 大盘宽基(沪深300/上证50) | 1–2% | 3–4% | 510300 |
| 高波动宽基(创业板/科创50) | 2–3.5% | 4–5% | **159915** |
| 行业主题(芯片/半导体/新能源/医药) | 2.5–4% | 5–6% | **159801** |
| 强风口主题ETF | 4–6% | 6–8% | 风口期 |
| 蓝筹个股 | 3–6% | 5–8% | 600276 |
| 活跃题材股 | 5–8% | 8–12% | — |

**铁律①: ETF止损<2%必然被洗。** 日均振幅2–3.5%的品种, 2%止损完全落在日内噪音区间内, 数学上注定高频误触发。

**铁律②: ETF用ATR止损优先。** 公式: `止损价 = 入场价 − max(2%, 2×ATR14)`。对159915, ATR14约0.08–0.10元(约2%), 2×ATR≈4%, 正好落在4%止损线; 159801风口期ATR放大, 止损自动放宽到5–6%, 避免风口洗盘。

**铁律③: 结构位止损做最终防线。** 跌破前日最低价**且当日收阴确认**, 或跌破20日均线收盘, 才触发硬止损。只触及前日低日内反弹=假破位, 不止损。

## 二、减仓节奏与时机

### 分批减仓的正确姿势

**核心原则: 减在急拉, 不减在回踩; 强势留底仓, 弱势才清仓。**

- 强势品种分 **3–4批**: 首批30%、二批30%、三批30%、**留底仓≥10%–30%**。
- 减仓触发位: 反弹至前高/阻力位/涨3%+ 急拉时减, 不是回踩-2%~-3%时减。回踩是加仓位不是减仓位。
- **强势板块底仓铁律**: 当日板块涨幅前20%或持仓站上MA10/MA20的强势ETF, **底仓留≥30%**吃主升浪, 用移动止盈(破MA10减第三批、破MA20清底仓)代替一刀切。
- **绝对禁止**: 把强势ETF清仓到0。趋势没走完清光=放弃尾部利润, 这是CFO今日159801第二刀的核心错误。

### 日内减仓时机 (A股T+1特性)

| 时段 | 特性 | 减仓策略 |
|---|---|---|
| 09:30–10:00 | 隔夜情绪宣泄, 波动最大, 急杀/V回高频 | **禁止减仓**(除非已破结构位止盈) |
| 10:00–11:30 | 情绪释放趋稳, 方向初现 | **最佳减仓窗口①** |
| 13:00–14:00 | 多平淡, 量缩 | 一般 |
| 14:00–14:30 | 尾盘方向选择, 主力意图显现 | **最佳减仓窗口②** |
| 14:30–14:57 | 异动多, 机构调仓 | 谨慎, 止盈兑现可 |
| 14:57–15:00 | 收盘集合竞价 | 流动性差, 避免大单 |

**铁律④: 不在早盘前30分钟减仓。** 早盘是情绪极端区, 急杀低点和急拉高点都常被随后的V回/回落吞噬。等10:00后再动。

## 三、日内时机判断

### A股T+1下的关键认知

T+1意味着**今天买的今天不能卖**, 所以"盘中急杀止损"卖的是昨天及之前的仓位。这给了清醒判断的时间窗口——**不急于即时反应**。

### 日内低点 vs 趋势破位

| 信号 | 日内低点(洗盘, V回概率高) | 趋势破位(真跌, 该减) |
|---|---|---|
| 成交量 | 杀跌量脉冲后**快速缩量** | 持续放量阴跌 |
| 分时量价 | 下跌无量、反弹放量(背离) | 反弹无量、下跌放量 |
| 结构位 | 触及前日低/均线**未实体跌破** | 跌破前日低/MA20**且收阴确认** |
| 板块联动 | 个别极端偏离后回归 | 板块**联动齐跌** |
| 触发后 | 缩量企稳、再杀不动 | 反弹无力、**再创新低** |
| 时点 | 早盘09:30–10:30急杀 | 14:00后破位(尾盘破位更真) |

**铁律⑤: 早盘急杀大概率V回, 不立即卖。** A股早盘急杀后V型反转是高频形态(隔夜情绪宣泄+主力低吸)。先看是否破结构位: 没破=噪音持有; 破了, 等反弹至-2%~-3%再减, 不在低点砍。

**V型反转识别**: 急跌后量能骤缩+分时现底背离+触及整数关/前低/均线支撑+指数同步企稳 → V回信号。此时是**加仓位不是减仓位**。

### 卖出决策树 (盘中急杀时)

```
急杀发生
 ├─ 是否在09:30-10:00早盘? → 是: 等10:00后判断, 不立即卖
 ├─ 是否跌破前日最低价实体?
 │   ├─ 否: 噪音, 持有(或V回时加仓)
 │   └─ 是: 是否收阴确认/板块联动下跌?
 │       ├─ 否(尾盘拉回): 假破位, 持有
 │       └─ 是: 等反弹至-2%~-3%再减, 分批
 └─ 是否跌破MA20收盘? → 是: 硬止损减仓
```

## 四、情绪管理与规则化

### Mark Douglas《交易心理分析》核心

**五大基本事实**: ① 任何事情都可能发生; ② 赚钱不必知道下一步; ③ 优势的盈亏呈现随机分布(单次随机, 长期才显优势); ④ 优势只表明概率高低, 非确定性; ⑤ 市场每一刻独一无二(过去≠未来, 不因记忆恐惧)。

**七项一致性原则**: 客观确认优势 → 预先界定风险 → 完全接受风险(否则不交易) → 按优势行动无犹豫 → 赚钱时付给自己 → 监测犯错可能 → **原则绝不可违背**。

**核心转化**: 把"我必须正确"的自我需求, 替换为"概率思维"。**恐惧来自未真正接受风险**——一旦1R风险预先界定并接受, 亏损就只是概率成本, 不是情绪事件。规则化交易让执行不依赖情绪状态。

### 恐慌性减仓的识别与规避

**恐慌信号(自查)**: 心跳加速/手抖/频繁刷新行情/想"先卖了再说"/无视止损位乱砍/在早盘前30分钟操作/卖了之后立刻想追回——出现任一条, **强制冷静30分钟, 不操作**。

**规则化替代人脑**:
- 所有减仓/止损**必须由rules.yaml触发或满足预设清单**, 不允许"看着盘面觉得要跌"就卖。
- 盘中只执行**已写好的计划**, 不做即兴决策。即兴=情绪=亏损源。
- 把"该不该卖"从盘中(高压)移到盘前(低压)——盘前定好计划, 盘中只执行。

## 五、今日实战对照与修正

### 错1: 159915在-1.66%(4.145)浅止损, 日内V回4.236, 踏空

**错在哪**:
1. 高波动宽基ETF用了<2%浅止损, 落在日均振幅(今日3.9%)内, **数学上注定被洗**。
2. 卖在日内低点(早盘急杀谷底), 没等反弹。
3. 4.145未跌破前日低/MA20/任何结构位, 纯粹被日内噪音吓出。

**正确做法**:
- 159915止损阈值≥4%(高波动宽基), 或 `入场价−max(2%,2×ATR14)` ≈ 4.215−0.085×2 ≈ 4.04。今日最低4.145根本未触及。
- 结构位止损: 跌破前日最低价且收阴才动。今日未破, 持有。
- 早盘急杀不卖, 等14:00后判断。
- **结论: 今日159915不该止损, 持有。** 浅止损规则本身是bug, 必须改。

### 错2: 159801第一刀1.615(对), 第二刀1.545偏早, 卖后涨5.7%

**错在哪**:
1. 第二刀减在回踩(1.545相对1.615回撤4.3%, 属正常震荡), **不是减在急拉**。回踩是加仓位。
2. 把2000股清仓到0, **强势板块(芯片当日强势)不留底仓**, 放弃尾部利润。
3. 第二刀应在反弹至阻力位(1.60附近靠近前高)再减, 或用移动止盈(破MA10才减第三批)。

**正确做法**:
- 减仓节奏: 首批30%(急拉位减, 1.615✓)、二批30%(等反弹阻力位或破MA10)、三批30%(破MA20)、**留底仓≥30%(600股)**。
- 1.545回踩不卖, 等反弹至1.58–1.60再减第二刀。
- 159801当日强势, 底仓至少留600股吃主升。

### 错3: 盘中急杀恐慌卖出, 没等反弹

**错在哪**: 情绪化执行, 违反Mark Douglas"完全接受风险、按优势行动无犹豫"。急杀时卖=卖在最差流动性和最差价格。把"该不该卖"留到盘中高压时刻即兴决定。

**正确做法**:
- 急杀不立即卖, 走第三节决策树: 早盘急杀→等10:00后; 看是否破结构位; 破了等反弹-2%~-3%再减。
- 盘前定好计划(哪只破什么位减多少), 盘中只执行, 不即兴。
- 出现恐慌信号(心跳/手抖/想"先卖再说")→**强制冷静30分钟不操作**。
- 159915今日正是典型早盘急杀V回, 按规则本该持有。

## 六、项目落地建议 (rules.yaml)

现有格式: `condition{field,op,value}` + `action` + `shares` + `max_trigger_per_day` + `cooldown_min` + `note` + `extra{time_after}`。建议新增/修改:

### 1. 修正159915止损(替换现有"破4.00减1000")

```yaml
- name: "创业板ETF结构止损-破前日低收阴"
  code: "159915"
  condition: { field: "price", op: "<=", value: 4.04 }  # 入场4.215-2*ATR(0.085)≈4.04
  action: "reduce"
  shares: 500
  max_trigger_per_day: 1
  cooldown_min: 0
  active: true
  note: "ATR结构止损(4%), 破前日低且收阴才减500, 留底仓"
  extra: { time_after: "14:00" }  # 避开早盘急杀, 尾盘判断
```

### 2. ETF浅止损护栏(新增, 防错1复发)

```yaml
- name: "ETF浅止损护栏-勿在日内低点卖"
  condition: { field: "change_pct", op: "<=", value: -2 }
  action: "info"
  max_trigger_per_day: 2
  cooldown_min: 30
  active: true
  note: "ETF跌2%仅属日内噪音, 未破前日低/MA20勿浅止损卖在低点"
  applies_to: "all_holdings"
  extra: { time_before: "10:00" }  # 早盘触发更需冷静
```

### 3. 早盘急杀冷静提醒(新增, 防错3复发)

```yaml
- name: "早盘急杀冷静-勿恐慌卖"
  condition: { field: "change_pct", op: "<=", value: -3 }
  action: "info"
  max_trigger_per_day: 1
  cooldown_min: 60
  active: true
  note: "早盘急杀大概率V回, 等14:00后判断是否破结构位, 不在低点砍"
  applies_to: "all_holdings"
  extra: { time_after: "09:30", time_before: "10:30" }
```

### 4. 强势ETF底仓保护(需monitor支持板块涨幅判断)

减仓规则的`shares`字段对强势品种加护栏: 当持仓ETF当日板块涨幅进前20%时, 单次reduce上限=持仓×70%, 强制留≥30%底仓。可作为`self_review.py`门禁: reduce日志若把强势ETF清仓到0 → 拦截警告。

### 5. 减仓时间窗门禁(防错2/错3的时机错)

给所有reduce类规则统一加 `extra: { time_after: "10:00", time_before: "14:50" }`, 从规则层禁止早盘前30分钟和尾盘集合竞价的减仓操作。

### 6. ATR止损工具支持(增强项)

`ohlcv.py`已能取日K, 新增helper `atr(code, period=14)`, 让monitor在评估止损时按 `入场价−max(2%,2×ATR)` 动态计算结构止损价, 替代固定百分比。这样不同ETF、不同波动期自动适配。

---

**一句话总结**: ETF止损<2%是数学bug不是策略; 减在急拉不减在回踩, 强势留≥30%底仓; 早盘急杀大概率V回, 先看结构位再决定; 所有卖出来自rules.yaml不是盘中情绪。
