## 提炼的铁律 (跨文档汇总)

### 止损
- **ETF止损<2%是数学bug** — 日均振幅2-3.5%, 浅止损注定被洗。高波动宽基≥4%, 行业ETF≥5%
- ETF用ATR止损: `入场价 − max(2%, 2×ATR14)`
- 结构位止损做最终防线: 跌破前日低**且收阴**才动

### 减仓
- **减在急拉, 不减在回踩**; 强势板块留底仓≥30%
- 不在早盘前30分钟(09:30-10:00)减仓 — 情绪极端区
- 早盘急杀大概率V回, 先看是否破结构位, 没破=噪音持有

### 成本核算
- **报任何盈亏/成本前, 必查decisions表buy行的price** — 不查db报数=瞎猜=禁止
- 部分减仓不改变剩余持仓成本, 只减数量 (lot制, parent_id关联)
- PROJECT_STATE.md的成本列需与db校验, 防文档成本≠db成本

### 技术面
- 量在价先。technical_scorer已加载volume却未用=最大缺口
- 缩量破位=洗盘(恒瑞06-26量比0.90x); 放量破位=真破位
- MA60是A股中线分水岭, 站上=多头, 破=转弱
- 回踩确认是加仓点不是卖点 (格兰维尔第二买点)

### 入场框架: 强势 vs 弱势 (audit 🟡3, 呼应假设[A])
两套框架分清, 别混用:
- **弱势票** (消费50/医药50这类下行趋势) → 等支撑回踩企稳: 跌到关键支撑位 + 缩量 + 不破 = 买点. 左侧抄底.
- **强势票** (通信ETF/创业板这类上行趋势, VCP高分) → 形态确认即入, 不死等深回踩: 平开/微高开+缩量=试仓1/3, 突破前高放量=加1/3, 日内浅回踩不破开盘价=补1/3. 止损=跌破当日开盘价或前低. 右侧追势.
- 判断强弱: 趋势方向(MA5/10/20排列) + VCP得分 + 量价. 强势票的"回踩"往往只是平开/日内浅探, 不是深跌.
- ⚠️ 反例教训: 06-30 515880通信ETF强势, 用弱势"等回踩1.70"框架 → 全天最低1.714没到 → 漏车+4.72%.

### 择时信号优先级 (audit 🔴2)
三信号打架时, 市场结构(market_regime派发日/FTD)为锚, 情绪(sentiment温度)为辅:
- NORMAL市场 → 可进攻 (开新仓OK)
- CAUTION + 情绪<40 → 防御 (不开新仓, 只减)
- CAUTION + 情绪≥40 → 中性
- HIGH/SEVERE → 防御 (只减不加, 考虑清仓)
- 实际涨幅不直接驱动决策 (滞后), 只用于事后印证结构信号是否失效.

### 止损体系层级 (audit 🟢5)
4套止损各司其职, 层级:
1. **rules.yaml硬触发** = 自动盯盘的即时止损线 (跌破即弹窗, 最优先)
2. **ATR结构止损** = 计划止损价, 写入plan_stop_loss, 供Portfolio Heat汇总
3. **Portfolio Heat** = 组合总风险Σ, 超6%减仓降敞口 (汇总层, 非单标的)
4. **MFE/MAE** = 事后验证止损宽度是否合理 (调参层, 攒样本后回测)
